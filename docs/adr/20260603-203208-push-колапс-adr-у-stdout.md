# Колапс `docs/adr/`-файлів у stdout команди `push`

**Status:** Accepted
**Date:** 2026-06-03

## Context and Problem Statement

Команда `push` друкує у stdout перелік змінених файлів (`git diff --cached --name-status`). Якщо в коміті присутні файли з `docs/adr/`, вони можуть займати десятки рядків, приховуючи значущі зміни.

## Considered Options

* Виводити кожен ADR-файл як окремий рядок разом із рештою
* Групувати усі `docs/adr/`-файли в один рядок із кількістю

## Decision Outcome

Chosen option: "Групувати `docs/adr/`-файли в один рядок із кількістю", because користувач явно попросив «docs/adr/ не показувались кожен в stdout а просто писалась їх кількість».

### Consequences

* Good, because stdout лишається компактним; перевірено симуляцією — виводить `📄 docs/adr/: 10 файл(ів)` замість 10 рядків.
* Neutral, because інші підтеки `docs/` (не `adr/`) виводяться поштучно — transcript не містить вимоги колапсувати їх.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information

- Формат рядка-колапсу: `📄 docs/adr/: N файл(ів)`.
- Реалізація в `npm/push.js` (stdout-блок): `git diff --cached --name-status` → grep-фільтр `docs/adr/` → підрахунок через `grep -c`.
- Тест-інваріант у `npm/tests/push.test.mjs`.
