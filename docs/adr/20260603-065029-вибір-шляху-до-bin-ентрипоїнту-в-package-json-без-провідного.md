## ADR Вибір шляху до bin-ентрипоїнту в `package.json` без провідного `./`

## Context and Problem Statement

При `npm publish` пакет `@7n/7` (тепер `@7n/n`) отримував попередження `"bin[7n]" script name bin/cli.js was invalid and removed`. Запис `bin` у `package.json` використовував префікс `./` (`"./bin/cli.js"`), що спричиняло авторемовлення під час серверної нормалізації npm.

## Considered Options

* Залишити `"./bin/cli.js"` — шлях із провідним `./`
* Прибрати `./` → `"bin/cli.js"` — відносний шлях без провідного слешу
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome

Chosen option: "Прибрати `./` → `"bin/cli.js"`", because локальний експеримент у `mktemp`-директорії довів: з `"./bin/cli.js"` `npm publish --dry-run` видаляє запис навіть для стандартного імені команди `cli`; без `./` попередження зникають.

### Consequences

* Good, because `npm publish --dry-run` проходить без рядків `invalid` / `removed`; `bin/n.js` потрапляє в тарбол і команда `n` доступна після глобального встановлення.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information

Файл: `npm/package.json`, поле `bin`.
Команди для перевірки: `npm publish --dry-run`, `npm pkg get bin`.
Емпіричний тест проводився у `mktemp`-директорії з перебором значень `"./bin/cli.js"` vs `"bin/cli.js"` vs `"cli": "./bin/cli.js"`.

## ADR Вибір назви npm-пакета: `@7n/n` замість `@7n/7`

## Context and Problem Statement

Перша спроба `npm publish` пакета `@7n/7` завершилась помилкою `E400 400 Bad Request — That word is not allowed`, бо npm денлист відхилив ім'я пакета `7`. Треба було обрати валідну назву під скоупом `@7n`.

## Considered Options

* `@7n/7` — оригінальна назва (відхилена денлистом npm)
* `@7n/cli` — описова назва за призначенням (`cli`)
* `@7n/n` — коротка літерна назва, одна буква
* `@vitaliytv/n7` — під власним нік-скоупом (згадано як запасний варіант)

## Decision Outcome

Chosen option: "`@7n/n`", because `@7n/n` вільне на реєстрі (E404), org `@7n` належить користувачу, і `npm publish` (`@7n/n@0.1.0`) пройшов успішно без помилок денлиста.

### Consequences

* Good, because `@7n/n@0.1.0` опубліковано публічно, команда доступна як `n` після `npm i -g @7n/n`.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information

Скоуп `@7n` — активна npm-організація, підтверджена на скріншоті в transcript.
`npm view @7n/n version` повертав E404 перед публікацією → ім'я було вільне.
Фінальна публікація: `+ @7n/n@0.1.0`.

## ADR Вибір імені команди та bin-файлу: `n` / `bin/n.js` замість `cli` / `bin/cli.js`

## Context and Problem Statement

Після публікації `@7n/n@0.1.0` з командою `cli` → `bin/cli.js` вирішено перейменувати ентрипоїнт, щоб ім'я команди (`n`) відповідало назві пакета (`@7n/n`) і bin-файл мав відповідне ім'я `n.js`.

## Considered Options

* `"cli": "bin/cli.js"` — команда `cli`, файл `cli.js` (опубліковано в 0.1.0)
* `"n": "bin/n.js"` — команда `n`, файл `n.js`
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome

Chosen option: "`"n": "bin/n.js"`", because користувач явно вказав замінити `"cli": "bin/cli.js"` на `"n": "bin/n.js"`, і `npm publish --dry-run` пройшов чисто з новим ім'ям.

### Consequences

* Good, because ім'я команди `n` узгоджене з назвою пакета `@7n/n`; `npm publish --dry-run` не містить попереджень.
* Bad, because версія `0.1.0` вже опублікована з командою `cli` — нова команда `n` потрапить лише у `0.1.1` і вище (незворотна особливість npm).

## More Information

Файл: `npm/package.json`, поля `bin` та `scripts.start`.
Фізичне перейменування: `npm/bin/cli.js` → `npm/bin/n.js` (через `mv`, оскільки файл не був у git-індексі).
Change-файл для наступного релізу: `npm/.changes/1780458577813-a40c6e.md` (`bump: patch`, `section: Changed`).
