## Поточна реалізація: «origin-дельта → локальний HEAD»

```
СТАН ДО:           СТАН ПІСЛЯ:
HEAD (локальні     HEAD (не рухається)
коміти + worktree) worktree = HEAD + origin-дельта (unstaged)
origin/branch (не чіпається)
```

**Алгоритм у фолбеці (FF неможливий):**
```
merge-base = ancestor(HEAD, origin/<branch>)
delta      = diff(merge-base..origin/<branch>)  ← що нового на origin
apply delta onto HEAD worktree                  ← unstaged
```

**Що маєш на виході:**
- HEAD = твої локальні коміти (не змінився)
- worktree = твоя робота + origin-зміни як unstaged
- потрібно вручну `git commit` щоб «злити» результат

## Пропозиція: «local-дельта → origin HEAD» (reverse-delta)

```
СТАН ДО:           СТАН ПІСЛЯ:
HEAD (локальні     HEAD = origin/<branch> (переміщено!)
коміти + worktree) worktree = HEAD + local-дельта (unstaged)
origin/branch = HEAD (збіглись)
```

**Алгоритм у фолбеці:**
```
backup = sha(HEAD) + git stash create (uncommitted)
git reset --hard origin/<branch>               ← HEAD переміщується на origin
local_base = ancestor(backup, origin/<branch>)
delta      = diff(local_base..backup)           ← що є локального
apply delta onto origin worktree               ← unstaged
```

**Що маєш на виході:**
- HEAD = origin (вже «знято» все чуже)
- worktree = твоя локальна робота як unstaged
- потрібно вручну `git commit` щоб «злити» результат

## Таблиця відмінностей

| | Поточна (origin→local) | Reverse-delta (local→origin) |
|---|---|---|
| HEAD після pull | не рухається | = origin (reset --hard) |
| Локальні коміти після | лишаються в HEAD | розпаковані у diff (uncommitted) |
| Blast radius при збої | нульовий — worktree не торкнуто | середній — HEAD перезаписано, відкат через reflog/backup |
| Семантика під `push` | push сквошить, але локальні коміти залишились паралельно | ідеальна: HEAD=origin, робота зверху unstaged, `push` одразу чистий |
| Якість резолву конфліктів | `_n7merge_delta` (mergiraf+агент) | той самий `_n7merge_delta`, але local-delta вхід |
| Складність реалізації | вже є | треба додати backup + reverse-направлення |
| Поведінка = `git rebase` (semantic) | ні — нагадує `merge --no-ff` unstaged | так — практично `git rebase origin/<branch>` без rebasing комітів |

## Практичний вибір

**Якщо ти переважно `push`-аєш після `pull`** — reverse-delta краще: HEAD вже = origin, worktree = твоя робота, `push` працює «чисто».

**Якщо `pull` — це «подивитись що прийшло, вирішити потім»** — поточна краще: безпечніша (HEAD не рухається), простіша, і `_n7merge_delta` вже стабільно перевірений у цьому напрямку.
