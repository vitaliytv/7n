## ADR Відображення часу створення та останньої зміни worktree у fzf-інтерфейсі `getw`

## Context and Problem Statement
У `fzf`-інтерфейсі `getw` (`npm/getw.js`) потрібно відображати часові мітки worktree, щоб користувач міг оцінити актуальність кожного worktree без виходу з меню. Потрібно було вирішити, як вимірювати дві різні події: момент створення worktree та момент реальної активності в ньому.

## Considered Options
* Birth time директорії (`stat -f '%SB'`) як сигнал часу створення worktree
* mtime найновішого файлу в директорії (виключаючи `.git/` та `node_modules/`) як сигнал "останньої зміни"
* mtime самої директорії worktree як сигнал "останньої зміни" (відхилений — не оновлюється при редагуванні існуючих файлів)

## Decision Outcome
Chosen option: "birth time директорії для створення + mtime найновішого файлу для зміни", because birth time директорії відповідає моменту `git worktree add`, а mtime директорії не відображає реальну активність при редагуванні файлів усередині неї.

### Consequences
* Good, because transcript фіксує очікувану користь: `📅 Створено` показує точний момент `git worktree add`, а `✏️ Змінено` — реальну активність у worktree.
* Bad, because birth time (`stat -f '%SB'`) є macOS-специфічним; на інших платформах або файлових системах без birth time хелпер `_getw_created` падає на mtime, що може дати неточний результат.

## More Information
- Змінений файл: `npm/getw.js`
- Нові хелпери: `_getw_created` (використовує `stat -f '%SB' -t '%Y-%m-%d %H:%M'` з fallback на `%Sm`) та `_getw_modified` (використовує `find … | xargs -0 stat -f '%m %N' | sort -rn | head -1` для пошуку найновішого файлу, fallback на mtime директорії)
- Фільтри при пошуку найновішого файлу: `-not -path '*/.git/*'` та `-not -path '*/node_modules/*'`
- Формат рядка fzf-елемента після змін: `<назва worktree>\n   Задача: <опис>\n   📅 Створено: YYYY-MM-DD HH:MM\n   ✏️ Змінено:  YYYY-MM-DD HH:MM`
- Парсинг вибраного worktree (`wt_name=${selected%%$nl*}`) не змінювався — лише перший рядок (назва) використовується для зворотного мапінгу
