## ADR Формат шляху `bin` у `package.json` без префікса `./`

## Context and Problem Statement
Під час `npm publish` з'являлося попередження `"bin[7n]" script name bin/cli.js was invalid and removed`, і bin-запис мовчки видалявся з пакету. Початково підозрювалося ім'я команди чи назва файлу, але реальна причина виявилася в іншому.

## Considered Options
* Залишити `"./bin/cli.js"` (як було спершу) — bin-запис відкидається npm при publish
* Використати `"bin/cli.js"` (без `./`) — bin-запис проходить нормалізацію чисто

## Decision Outcome
Chosen option: `"bin/cli.js"` без префікса `./`, because емпіричне тестування в ізольованому temp-пакеті довело, що npm під час `publish` відкидає будь-який bin-запис із `./`-значенням незалежно від імені команди чи назви файлу — навіть ключ `"cli": "./bin/cli.js"` видалявся, тоді як `"7n": "bin/cli.js"` проходив чисто.

### Consequences
* Good, because `npm publish --dry-run` більше не видає `auto-corrected`/`invalid`/`removed` для bin-запису; `bin/cli.js` відображається в `npm notice` і потрапляє в пакет.
* Bad, because `npm pkg fix` цю проблему **не** фіксує (лишає `./` як є), тому його порада в тексті попередження вводить в оману — треба знати про правило вручну.

## More Information
* Файл: `npm/package.json`, поле `bin`
* Фінальний стан: `"cli": "bin/cli.js"` (без `./`)
* Перевірка: `npm publish --dry-run 2>&1 | grep -iE "invalid|removed|auto-corrected"`
* `npm pkg fix` не є надійним інструментом для виявлення цієї помилки — transcript підтверджує.

## ADR Перейменування bin-файлу та команди в `cli`

## Context and Problem Statement
Оригінальний bin-файл називався `bin/7.js`, а команда в `package.json` — `"7"`. Під час розгляду npm-попередження виникло питання про назву і файлу, і команди; користувач вирішив привести їх до однакового й описового імені.

## Considered Options
* `"7"` / `bin/7.js` — оригінальна назва
* `"7n"` / `bin/7.js` — проміжний варіант (обраний як варіант 2 з пропозицій асистента)
* `"cli"` / `bin/cli.js` — фінальний варіант, запропонований і затверджений користувачем

## Decision Outcome
Chosen option: `"cli": "bin/cli.js"`, because користувач явно запросив спочатку перейменування файлу на `cli.js`, а потім — ключа команди на `cli`; підтверджено `npm publish --dry-run` без попереджень.

### Consequences
* Good, because назва файлу і команди тепер збігаються та є описовими (`cli`); `npm publish --dry-run` проходить чисто.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
* Файли: `npm/bin/cli.js` (перейменовано з `bin/7.js` через `mv`), `npm/package.json`
* Оновлені поля в `package.json`: `bin["cli"]`, `scripts.start`
* Попередній `scripts.start` вказував на `bun ./bin/7.js` → оновлено до `bun ./bin/cli.js`
