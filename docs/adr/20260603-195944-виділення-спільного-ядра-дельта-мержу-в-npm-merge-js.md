## ADR Виділення спільного ядра дельта-мержу в `npm/merge.js`

## Context and Problem Statement
Механізм «інтелектуального» багатокрокового мержу (Tier 0–3: `git apply` → `git merge-file --diff3` → `mergiraf solve` → LLM-агент) існував монолітно в `getw.js`. Потрібно було створити команду `getpull` із тією самою логікою, але іншим джерелом (`origin/<гілка>` замість локального worktree).

## Considered Options
* Дублювати zsh-логіку в `getpull.js` окремо від `getw.js`
* Винести спільне ядро в окремий модуль `merge.js` і використовувати його з обох команд

## Decision Outcome
Chosen option: "Винести спільне ядро в окремий модуль `merge.js`", because дублювання tier-пайплайну ускладнило б підтримку і призвело б до розходження реалізацій.

### Consequences
* Good, because `getw.js` і `getpull.js` спільно використовують одну реалізацію `_n7merge_delta`, `MERGE_ZSH_LIB` та `runZsh` — зміна логіки тирів потребує правки в одному місці.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Файли: `npm/merge.js` (нові: `MERGE_ZSH_LIB`, `runZsh`), `npm/getw.js` (рефакторинг), `npm/getpull.js` (новий). `npm/package.json#files` доповнено `"getpull.js"`, `"merge.js"`. Тести: `npm/tests/merge.test.mjs` (новий), `npm/tests/index.test.mjs` (доповнений); 29/29 green.

## ADR Delta-мердж у `getpull` замість повного `git checkout origin/<br> -- .`

## Context and Problem Statement
Перша реалізація `getpull` виконувала `git checkout origin/$branch -- .` — повний checkout зрізу, що перезаписував **усі** tracked-файли версіями з origin. Після першого запуску користувач запитав, чи нічого не затерлось. Аналіз показав: тоді пронесло лише тому, що origin був на 3 коміти попереду без локальних правок поверх; але на брудному дереві команда затерла б незакомічені зміни.

## Considered Options
* Залишити `git checkout origin/<branch> -- .` (повний зріз)
* Застосовувати лише дельту `merge-base(HEAD, origin/<branch>)..origin/<branch>` через той самий Tier 0–3-пайплайн, що й у `getw`

## Decision Outcome
Chosen option: "Дельта через Tier 0–3-пайплайн", because той самий механізм вже реалізовано в `getw` і він safe на брудному дереві: переносяться лише зміни з origin, а не весь зріз, тому незакомічені правки tracked-файлів не затираються.

### Consequences
* Good, because transcript фіксує очікувану користь: staged-вміст і HEAD зберігаються за будь-яких обставин (підтверджено емпіричним тестом у `/tmp/gptest`); конфлікт лишає diff3-маркери замість тихого затирання.
* Good, because `getpull` повертає exit code `1` при незавершеному мержі — можна використовувати в CI/скриптах.
* Bad, because суто-unstaged вміст конфліктного файлу живе лише всередині маркерів після мержу — окремого rollback-артефакту немає (обговорення `git stash create`-знімку як pre-flight backup залишилось відкритим наприкінці сесії).

## More Information
Команди в тесті: `PATH="$ROOT/fakebin:$PATH" N7MERGE_NO_MERGIRAF=1 bun "$CLI" getpull main`. Файл `npm/getpull.js`: `git fetch origin <branch>` → `_n7merge_delta "HEAD" "origin/<branch>"`. Перевірка `zsh -n` обох скриптів — OK. Обговорення `git stash push` (drawbacks: руйнує staged/unstaged-розрізнення, виносить конфлікт поза tier-пайплайн) vs `git stash create` (знімок без очищення дерева) лишилось відкритим — код не змінювався.

## ADR Нейтральні env-назви `N7MERGE_*` з backward-compat фолбеком на `GETW_*`

## Context and Problem Statement
Після рефакторингу `getw` у `merge.js` env-змінні управління мерджем (`GETW_MERGE_MODEL`, `GETW_MERGE_CURSOR_MODEL`, `GETW_NO_MERGIRAF`) мали префікс `GETW_`, хоча ядро тепер спільне для `getw` і `getpull`. Користувач явно попросив нейтральний префікс.

## Considered Options
* Залишити `GETW_*`-префікс
* Перейти на нейтральні `N7MERGE_*` з backward-compat вкладеним фолбеком `${N7MERGE_X:-${GETW_X:-default}}`

## Decision Outcome
Chosen option: "`N7MERGE_*` з фолбеком на `GETW_*`", because `GETW_`-префікс відображає лише одну команду, хоча ядро спільне; фолбек не ламає наявних конфігів користувачів.

### Consequences
* Good, because transcript фіксує очікувану користь: пріоритет `N7MERGE_` → `GETW_` → дефолт підтверджено в zsh (`zsh -c '...'`) — `N7MERGE_MODEL=haiku` виграє над `GETW_MERGE_MODEL=opus`.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Файл `npm/merge.js`: змінна `${N7MERGE_MODEL:-${GETW_MERGE_MODEL:-sonnet}}`, `${N7MERGE_CURSOR_MODEL:-${GETW_MERGE_CURSOR_MODEL:-...}}`, `${N7MERGE_NO_MERGIRAF:-${GETW_NO_MERGIRAF:-0}}`. Тест у `npm/tests/merge.test.mjs`: `'env-кнопки нейтральні (N7MERGE_*) із backward-фолбеком на GETW_*'`. Коментарі та JSDoc у `getw.js`/`getpull.js`/`README.md` оновлено відповідно.
