# getw: інтелектуальний мерж конфліктів через LLM-агента

**Status:** Accepted
**Date:** 2026-06-03

## Context and Problem Statement

Після переходу `getw` на дельта-перенесення через `git apply` при конфлікті команда зупинялась з помилкою та зберігала worktree для ручного розв'язання. Потрібна була автоматична розв'язка конфліктів без ручного втручання.

## Considered Options

- Падати при конфлікті, зберігати worktree для ручного розв'язання (попередня поведінка після першого виправлення)
- `git apply --3way` з подальшим викликом LLM-агента (`claude -p`, фолбек `cursor-agent -p`) для розв'язання конфліктних маркерів

## Decision Outcome

Chosen option: "`git apply --3way` + LLM-агент (`claude -p` / `cursor-agent -p`)", because це дозволяє зберегти дані обох сторін у вигляді конфліктних маркерів і делегувати інтелектуальне злиття агенту без втрати змін; worktree видаляється лише після підтвердженого чистого результату.

### Consequences

- Good, because при конфлікті втрата даних не відбувається — worktree зберігається до успішного або явно підтвердженого ручного завершення.
- Good, because пріоритетний вибір CLI (`claude` → `cursor-agent`) відповідає конвенції LLM-хуків проєкту; моделі налаштовуються через змінні середовища.
- Bad, because `--3way` потребує, щоб blob-об'єкти merge-base були доступні у репозиторії; якщо жодного з CLI немає в `PATH` або агент не прибирає всі маркери — worktree зберігається і потрібне ручне доведення.
- Bad, because `claude -p` має бути авторизований у середовищі виконання — без цього агентська гілка недоступна.

## More Information

- Файли: `npm/getw.js` (helper `_getw_resolve_with_agent`, рядки 9–31 і 93–155), `npm/README.md`.
- Алгоритм: 1) `git apply` (без `--3way`) — якщо чисто, кінець; 2) при конфлікті — `git apply --3way --whitespace=nowarn`; 3) знайти конфліктні файли через `git diff --name-only --diff-filter=U`; 4) викликати `_getw_resolve_with_agent`: спершу `claude -p`, фолбек `cursor-agent -p`; 5) перевірити відсутність маркерів `<<<<<<<` / `>>>>>>>`; 6) `git reset` → worktree видаляється.
- Команди: `git apply --3way --whitespace=nowarn`, `claude -p "$prompt" --permission-mode acceptEdits --model "${GETW_MERGE_MODEL:-sonnet}"`, `cursor-agent -p --force --output-format text --model "${GETW_MERGE_CURSOR_MODEL:-claude-4.6-sonnet-medium}"`.
- Конфігурація моделей через змінні середовища: `GETW_MERGE_MODEL` та `GETW_MERGE_CURSOR_MODEL`.
- Перевірка відсутності маркерів: `grep -rq '<<<<<<<\|>>>>>>>' $files`.
- Change-файл: bump `minor`, секція `Added`.
- Синтаксис zsh перевірено через `zsh -n`.
