## ADR Стратегія мерджу: `git checkout <ref> -- .` + `git reset .` як unstaged

## Context and Problem Statement
Потрібно перенести зміни з git-ref (worktree або `origin/<гілка>`) у поточне робоче дерево так, щоб результат лишався **unstaged** — без авто-merge-коміту і без конфліктних маркерів.

## Considered Options
* `git checkout <ref> -- .` + `git reset .` (повне накочування файлів ref → unstaged)
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "`git checkout <ref> -- .` + `git reset .`", because це вже існуючий підхід у `getw.js`, який користувач назвав «інтелектуальним мерджем», і нова команда `getpull` використовує той самий механізм через `mergeRefIntoWorktree`.

### Consequences
* Good, because результат — чисті unstaged-зміни, які розробник переглядає й комітить вручну.
* Bad, because якщо в робочому дереві є незакомічені зміни у файлах, що змінилися й у ref — `git checkout <ref> -- .` тихо перезапише їх (підтверджено в transcript після першого `getpull`-запуску; користувач запитав «чи нічого не затерлось»).

## More Information
Команда `npm/merge.js`: `mergeRefIntoWorktree(refExpr)` будує zsh-фрагмент із `git checkout "$refExpr" -- .` та `git reset .`. Ризик перезапису незакомічених змін зафіксований у transcript; в кінці сесії запропоновано додати `git status --porcelain` + early-exit як захист.

## ADR Нова команда `getpull` — мердж із `origin/<гілка>`

## Context and Problem Statement
Розробнику потрібно накочувати стан `origin/<гілка>` у поточне робоче дерево тим самим механізмом, що й `getw` використовує для локальних worktree.

## Considered Options
* `getpull [гілка]` — `git fetch origin <гілка>` + `mergeRefIntoWorktree('origin/<гілка>')`
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "`getpull [гілка]`", because користувач явно попросив «таку ж команду, але між поточним станом та `origin/назва_гілки`».

### Consequences
* Good, because transcript фіксує очікувану користь: успішний smoke-test `bun ./npm/bin/n.js getpull` підтягнув 3 коміти з `origin/main` без втрати даних.
* Bad, because транзитно перезаписує файли, що збігаються між HEAD і origin (ризик підтверджено в transcript).

## More Information
Файл: `npm/getpull.js`. Гілка — аргумент `argv[0]` або результат `git branch --show-current`; обробка detached HEAD і відсутнього remote-ref. Команда реєструється в `index.js` з DI-інжектом `io.getpull` для тестування. Тест: `npm/tests/index.test.mjs` (делегування getpull із назвою гілки).
