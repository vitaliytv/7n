## ADR Нова команда `getpull` — delta-мердж з `origin/<branch>`

## Context and Problem Statement
Перша реалізація `getpull` використовувала `git checkout origin/$branch -- .`, що перезаписує **весь зріз** цільової гілки, затираючи незакомічені правки tracked-файлів. Потрібна безпечна команда, яка переносить лише нові зміни з origin у поточне дерево.

## Considered Options
* Delta-мердж `merge-base(HEAD, origin/<branch>)..origin/<branch>` через те саме ядро Tier 0–3, що й `getw`
* `git checkout origin/$branch -- .` (повний зріз, перша реалізація) — відкинуто через деструктивність на брудному дереві
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Delta-мердж через спільне ядро Tier 0–3", because `git checkout -- .` затирає незакомічені правки; delta-підхід переносить лише дельту й залишає незачеплені файли недоторканими.

### Consequences
* Good, because transcript фіксує очікувану користь: перевірено емпірично на репозиторії — HEAD, staged (index) і незачеплені unstaged-файли збережено після `getpull`; конфліктний файл лишає маркери (exit 1), не затирає вміст.
* Bad, because суто-unstaged вміст конфліктного файлу живе лише всередині `<<<<<<<`-маркерів (окремої копії немає, бо unstaged не зафіксовано у `HEAD`).

## More Information
Команди в zsh-скрипті: `git fetch origin <branch>`, потім `_n7merge_delta "HEAD" "origin/<branch>"`. Гілка — `$1` або `git branch --show-current`. Інтеграція: `npm/index.js` (DI `io.getpull`), `npm/types/index.d.ts`, `npm/README.md`. Валідовано: `zsh -n`, `vitest` 31/31, `@nitra/cursor fix changelog` 1/1.

## ADR Pre-flight бекап незакомічених змін через `git stash create`

## Context and Problem Statement
Tier-3 (LLM-агент) редагує конфліктні файли в робочому дереві. Якщо агент або будь-який інший Tier пошкодить файл, незакомічений вміст (unstaged/staged) буде втрачено без точки відкату.

## Considered Options
* `git stash create` + `git stash store` (знімок-коміт, **дерево не чіпає**)
* `git stash push` (очищає дерево) — відхилено: втрачає staged/unstaged-розрізнення, переносить конфлікт на `stash pop` повз Tier-пайплайн, залишає лінгерінг-stash при конфліктному `pop`
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "`git stash create` + `git stash store`", because `git stash push` руйнує саме ті гарантії (збереження index, прямий мердж через tiers), які є головною перевагою delta-підходу; `create` робить знімок не чіпаючи дерево.

### Consequences
* Good, because transcript фіксує очікувану користь: знімок реально містить і staged-, і unstaged-правки (перевірено `git show <sha>:file`); на чистому дереві `git stash create` нічого не створює — крок пропускається; в stdout друкується точна команда відновлення `git stash apply <sha>`.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Реалізовано в `npm/merge.js`, функція `_n7merge_delta` (перший крок перед `git merge-base`). Env-контроль не передбачено — знімок завжди. SHA бекапу виводиться у stderr/stdout зі `🛟` emoji. Тести: `npm/tests/merge.test.mjs` (перевірка виклику `git stash create`, `git stash store`, наявності `git stash apply <sha>` у повідомленні).
