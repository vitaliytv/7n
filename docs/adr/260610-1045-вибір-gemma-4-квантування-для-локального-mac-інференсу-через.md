## ADR Вибір Gemma 4 квантування для локального Mac-інференсу через omlx

## Context and Problem Statement

Користувач запустив локальний LLM inference server `omlx` v0.4.3 (Homebrew, Apple Silicon, 16 GB unified memory) і хотів порівняти варіанти квантування Gemma 4 для подальшої роботи. Початкова модель `rajaschitnis--gemma-4-12b-it-text-only-4bit-mlx` займала 10.7 GB і вимагала `chunked_prefill` через metal cap `~11.8 GB`; крім того, вона застрягала в reasoning loop при code-завданнях навіть з `max_tokens=2500`.

## Considered Options

* `rajaschitnis/gemma-4-12b-it-text-only-4bit-mlx` — ручне 4-bit квантування, 10.7 GB
* `mlx-community/gemma-4-e4b-it-OptiQ-4bit` — Gemma 4 E4B (MoE ~4B активних), 4-bit OptiQ, 7.5 GB
* `mlx-community/gemma-4-12B-it-OptiQ-4bit` — 12B, 4-bit OptiQ, 9.0 GB
* `mlx-community/gemma-4-12B-it-qat-4bit` — 12B, QAT (quantization-aware training), 12.4 GB

## Decision Outcome

Chosen option: **залежить від пріоритету** — два переможці за призначенням:

1. **`gemma-4-e4b-it-OptiQ-4bit`** для задач де потрібна швидкість, because 28 tps (3.5× швидше за 12B), поміщається без `chunked_prefill`, 4/4 тестів pass, відповідає без reasoning overhead.
2. **`gemma-4-12B-it-qat-4bit`** для задач де потрібен chain-of-thought, because це єдиний 12B-варіант де thinking-модель виходить з reasoning до `[answer]` (QAT зберігає точність активацій), 7.5 tps.

Обидва варіанти кращі за `rajaschitnis` vanilla, яка видаляється з системи.

### Consequences

* Good, because transcript фіксує очікувану користь: `rajaschitnis` видаляється (`unload` виконано), звільняється 10.7 GB RAM і 10.55 GB на диску.
* Good, because `12B-OptiQ` (9 GB, 8 tps, без `chunked_prefill`) залишається як збалансований варіант.
* Bad, because `12B-QAT` (12.4 GB) потребує `chunked_prefill` на 16 GB Mac при зайнятій системній RAM — memory ceiling залишається у `custom: 12.0 GB` у `~/.omlx/settings.json`.

## More Information

* Змінені параметри `~/.omlx/settings.json`: `memory_guard_tier: "custom"`, `memory_guard_custom_ceiling_gb: 12.0`, `chunked_prefill: true`, `auth.api_key: "omlx-local-test-key"`
* Бекап оригінального конфігу: `~/.omlx/settings.json.bak.1781072139`
* Модель `rajaschitnis` вивантажена з RAM через `POST /admin/api/models/{model_id}/unload`; видалення з `~/.cache/huggingface/hub/models--rajaschitnis--...` — через `huggingface-cli delete-cache` або `rm -rf`
* Всі тести (UA history, missing dollar, sheep trap, RLE encode Python) підтверджені програмно; `rle_encode` — 6/6 unit tests pass для OptiQ і QAT
