## Context and Problem Statement
У CLI `@7n/n` вже є `getw` і `getpull` для отримання змін. Відсутня симетрична команда для публікації: розробник хоче одним вчинком сквошити всі локальні коміти, отримати автоматично сформоване повідомлення у стилі Gitmoji + Monorepo і запушити результат.

* Назва `getpush`
* Назва `pushup`
* Назва `cmpush`
* Назва `push`

## ADR Автоматичний `getpull` при дивергенції замість зупинки

## Context and Problem Statement
Якщо `origin/<branch>` містить коміти, яких немає локально, squash поверх них зробив би push non-fast-forward або затер би віддалені зміни. Потрібно обробити цей випадок.

## Considered Options
* STOP з порадою запустити `npx @7n/n getpull` вручну
* Автоматично викликати `getpull` перед squash

## Decision Outcome
Chosen option: "Автоматично викликати `getpull`", because користувач явно вказав «замість STOP — автоматично getpull».

### Consequences
* Good, because transcript фіксує очікувану користь: команда залишається одним кроком без ручного втручання при дивергенції.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Логіка реалізована у zsh-скрипті `PUSH_ZSH_SCRIPT` всередині `npm/push.js` через виклик `_n7merge_delta` з `MERGE_ZSH_LIB` (`npm/merge.js`).

## ADR Відсутність інтерактивного підтвердження перед push

## Context and Problem Statement
Push — незворотна дія, що впливає на спільний remote. Під час планування розглядався варіант `[y/N]`-підтвердження з прапором `-y`/`--yes` для автоматизації.

## Considered Options
* Інтерактивний запит `[y/N]` з прапором `-y`/`--yes`
* Без підтвердження, лише stdout-вивід назви коміту та списку файлів

## Decision Outcome
Chosen option: "Без підтвердження, вивід у stdout", because користувач явно обрав «без питань, в stdout показити назву коміту та список файлів».

### Consequences
* Good, because transcript фіксує очікувану користь: команда придатна для non-interactive середовищ і скриптів.
* Bad, because Neutral, because transcript не містить підтвердження наслідку щодо ризику випадкового push.

## More Information
Вивід реалізований у zsh-скрипті `PUSH_ZSH_SCRIPT` в `npm/push.js`.

## ADR `--no-verify` для коміту у команді `push`

## Context and Problem Statement
У проєкті є pre-commit хуки (`hk`/changelog-перевірки), що запускаються при `git commit`. При автоматичному сквош-коміті ці хуки можуть блокувати публікацію або вимагати додаткового контексту, недоступного на момент зупинки.

## Considered Options
* Залишити хуки увімкненими
* `--no-verify` (пропустити хуки)

## Decision Outcome
Chosen option: "`--no-verify`", because користувач прямо вказав «хук для цього випадку не потрібен».

### Consequences
* Good, because transcript фіксує очікувану користь: автоматичний squash-коміт не блокується changelog-перевірками.
* Bad, because Neutral, because transcript не містить підтвердження наслідку щодо можливого обходу важливих перевірок.

## More Information
Прапор `--no-verify` застосований у `git commit` всередині `PUSH_ZSH_SCRIPT` в `npm/push.js`.

## ADR LLM-генерація Multi-line Gitmoji+Monorepo повідомлення коміту

## Context and Problem Statement
Команда `push` сквошує довільну кількість локальних комітів в один. Формулювати підсумкове повідомлення вручну — зайвий крок; існуючі команди `getw`/`getpull` вже використовують `claude -p` / `cursor-agent -p` для LLM-операцій.

## Considered Options
* Subject-рядок (однорядкове повідомлення)
* Multi-line повідомлення (subject + body)

## Decision Outcome
Chosen option: "Multi-line повідомлення через LLM", because користувач явно обрав Multi-line, а генерацію через агента вже передбачав план (патерн `_n7merge_resolve_with_agent` з `merge.js`).

### Consequences
* Good, because transcript фіксує очікувану користь: повідомлення відповідає стилю Gitmoji + Monorepo, українською, зі scope з changed-workspaces, без ручного введення.
* Bad, because Neutral, because transcript не містить підтвердження наслідку щодо якості повідомлень при відсутності LLM-агента.

## More Information
Модель визначається через `N7COMMIT_MODEL` → `N7MERGE_MODEL` → `sonnet` (аналогічно до `merge.js`). Вивід агента зберігається у tempfile, коміт виконується через `git commit -F msgfile --no-verify`. Реалізовано у `npm/push.js`.
