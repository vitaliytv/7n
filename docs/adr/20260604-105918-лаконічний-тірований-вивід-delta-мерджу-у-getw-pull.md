## ADR Лаконічний тірований вивід delta-мерджу у getw/pull

## Context and Problem Statement

`getw` і `pull` використовують спільне ядро `_n7merge_delta` (у `npm/merge.js`) для переносу дельти між гілками. При Tier 1 (clean git apply) та Tier 2 (mergiraf) вивід містив побічні повідомлення: `error: patch failed`, `error: … patch does not apply`, а також пофайлові `mergiraf розв'язав …`. Ці повідомлення лякали користувача навіть при успішному завершенні і не несли actionable інформації.

## Considered Options

* Показувати лаконічний тірований підсумок замість пофайлового шуму (подавити `error: patch failed` і per-file mergiraf-рядки; показувати деталі лише для Tier 3 LLM).
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome

Chosen option: "Показувати лаконічний тірований підсумок замість пофайлового шуму", because користувач явно вказав бажаний формат: рядки `error:` мають бути приховані, деталі mergiraf — заховані, а резюме має складатися з трьох ліній (Tier 1 / Tier 2 / Tier 3) і per-file блоків лише для Tier 3 (приймач, джерело, результат, LLM-коментар).

### Consequences

* Good, because transcript фіксує очікувану користь: користувач перестане бачити false-alarm `error:` повідомлення при успішному мерджі.
* Good, because Tier 3 тепер показує структуровані блоки конфлікту (ours / src / result) разом із LLM thinking, що дає контекст без шуму проміжних кроків.
* Bad, because transcript не містить підтверджених негативних наслідків.
