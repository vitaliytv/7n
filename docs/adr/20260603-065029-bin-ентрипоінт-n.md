# Перейменування bin-ентрипоінта та команди: `n` / `bin/n.js`

**Status:** Accepted
**Date:** 2026-06-03

## Context and Problem Statement

Після публікації `@7n/n@0.1.0` bin-команда в пакеті мала назву `cli` (файл `bin/cli.js`), що не відповідало назві пакета `@7n/n`. Вирішено привести ім'я команди і bin-файлу у відповідність до назви пакета.

## Considered Options

* `"cli": "bin/cli.js"` — команда `cli`, файл `cli.js` (опубліковано у v0.1.0)
* `"n": "bin/n.js"` — команда `n`, файл `n.js` (відповідає назві пакета `@7n/n`)
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome

Chosen option: "команда `n`, файл `bin/n.js`", because користувач явно вказав замінити команду `cli` на `n` і файл `cli.js` на `n.js`, і `npm publish --dry-run` пройшов чисто з новим ім'ям.

### Consequences

* Good, because ім'я команди `n` узгоджене з назвою пакета `@7n/n`; `npm publish --dry-run` проходить без попереджень.
* Bad, because версія `0.1.0` вже опублікована з командою `cli` — нова команда `n` потрапить лише у `0.1.1` і вище (незворотна особливість npm-реєстру).

## More Information

* Файл: `npm/package.json`, поля `bin` та `scripts.start`
* Фізичне перейменування: `npm/bin/cli.js` → `npm/bin/n.js` (через `mv`)
* Change-файл для наступного релізу: `npm/.changes/1780458577813-a40c6e.md` (`bump: patch`, `section: Changed`)
* Перевірка: `npm publish --dry-run 2>&1 | grep -iE "invalid|removed|warn"`
