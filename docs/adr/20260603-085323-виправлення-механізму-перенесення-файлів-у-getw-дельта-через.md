## ADR Виправлення механізму перенесення файлів у `getw`: дельта через `git apply` замість `git checkout -- .`

## Context and Problem Statement

Команда `getw` (у `npm/getw.js`) переносить зміни з git-worktree у поточну гілку, використовуючи `git checkout "$target_branch" -- .`. Це сліпо перезаписує **всі** файли робочого дерева вмістом worktree-гілки, через що файли, змінені лише у поточній гілці (але не в worktree-гілці), замінюються їх старими версіями з worktree. Зміни втрачаються без жодного попередження й без виявлення конфліктів.

## Considered Options

* Залишити `git checkout "$target_branch" -- .` (поточна поведінка)
* Переносити лише дельту worktree-гілки від спільного предка через `git diff --binary "$merge_base" "$target_branch" | git apply --whitespace=nowarn`

## Decision Outcome

Chosen option: "Переносити лише дельту через `git diff --binary | git apply`", because цей підхід чіпає виключно файли, які реально змінені у worktree-гілці відносно merge-base, а не перезаписує все дерево; при конфлікті `git apply` падає явно замість мовчазного затирання даних.

### Consequences

* Good, because файли, змінені **тільки** у поточній гілці, більше не перезаписуються старими версіями з worktree-гілки.
* Good, because при конфлікті перенесення зупиняється з помилкою, worktree зберігається для ручного злиття, а зміни поточної гілки не втрачаються.
* Good, because `git apply` без `--index` кладе зміни як unstaged — поведінка з точки зору користувача залишається незмінною.
* Bad, because transcript не містить підтверджених негативних наслідків.

* Змінений файл: `npm/getw.js`, рядки 64–78 (після правки).

* Прапор `--binary` у `git diff` — щоб коректно обробляти бінарні файли.
