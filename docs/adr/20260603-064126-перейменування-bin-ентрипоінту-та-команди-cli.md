## ADR Перейменування bin-ентрипоінту та команди CLI

## Context and Problem Statement
`npm publish` відхиляв bin-запис у `npm/package.json` з попередженням `"bin[7]" script name bin/7.js was invalid and removed`. Bin-файл `bin/7.js` та команда `"7"` потребували виправлення, щоб пакет публікувався із робочим CLI.

## Considered Options
* Лишити `bin/7.js`, змінити лише ключ команди на буквено-цифровий (`7n`, `n7`)
* Перейменувати файл `bin/7.js` → `bin/cli.js` і команду → `cli`
* Використати інше коротке ім'я (`_7`, `g7`, `seven`)

## Decision Outcome
Chosen option: "Перейменувати файл `bin/7.js` → `bin/cli.js` і команду → `cli`", because користувач явно вказав `"давай 7.js перейменуємо в cli.js"` і потім `"перейменуй в cli також"` для команди.

### Consequences
* Good, because `bin/cli.js` — описова назва, сумісна з усіма перевірками npm; `npm publish --dry-run` дав чистий вихід без жодних `warn`.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Змінені рядки в `npm/package.json`:
```json
"bin": { "cli": "bin/cli.js" },
"scripts": { "start": "bun ./bin/cli.js" }
```
Фізичне перейменування: `mv bin/7.js bin/cli.js`. Пошук по `"7\.js"` підтвердив, що більше ніяких посилань у репо не було.

## ADR Видалення префікса `./` у значенні `bin`

## Context and Problem Statement
Навіть після перейменування файлу та команди `npm publish --dry-run` продовжував видавати `"bin[7n]" script name bin/cli.js was invalid and removed`. Потрібно було знайти справжню причину відхилення bin-запису нормалізатором npm.

## Considered Options
* Продовжувати шукати проблему в імені команди (цифра, скоуп тощо)
* Перевірити значення шляху — з `./` та без

## Decision Outcome
Chosen option: "Прибрати префікс `./` зі значення `bin`", because емпіричний тест (temp-пакети з різними комбінаціями) довів: `"cli": "./bin/cli.js"` → `removed`; `"cli": "bin/cli.js"` → чисто. Саме `./` викликало відхилення, а не ім'я команди чи назва файлу.

### Consequences
* Good, because transcript фіксує очікувану користь: `npm publish --dry-run` після видалення `./` дав `npm notice 107B bin/cli.js` без жодного `warn`.
* Bad, because `npm pkg fix` не ловить цю проблему і залишає `./` як є, тому попередження npm (`"Please run npm pkg fix"`) вводить в оману.

## More Information
Формат після виправлення: `"bin": { "cli": "bin/cli.js" }`. Тест проводився через `npm publish --dry-run` у temp-директоріях з package.json-заготовками для варіантів `"./bin/cli.js"` та `"bin/cli.js"`.

## ADR Додавання `publishConfig.access: "public"` для scoped-пакету

## Context and Problem Statement
`npm publish` повернув `E402 Payment Required` для пакету `@7n/7`. npm вважає будь-який scoped-пакет (`@scope/...`) приватним за замовчуванням, що вимагає платного плану.

## Considered Options
* Передавати `--access public` щоразу при виклику `npm publish`
* Закріпити `publishConfig.access: "public"` у `package.json`

## Decision Outcome
Chosen option: "Закріпити `publishConfig.access: \"public\"` у `package.json`", because це зберігається в репо й не потребує запам'ятовування флага при кожній публікації (включно з CI).

### Consequences
* Good, because transcript фіксує очікувану користь: dry-run після зміни показав `Publishing to https://registry.npmjs.org/ with tag latest and public access (dry-run)`.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Блок у `npm/package.json`:
```json
"publishConfig": { "access": "public" }
```
Альтернатива разового виклику: `npm publish --access public`.

## ADR Перейменування пакета з `@7n/7` на `@7n/cli`

## Context and Problem Statement
`npm publish` повернув `E400 Bad Request — That word is not allowed` для імені `@7n/7`. Слово/токен `7` знаходиться в серверному денлисті npmjs.com; ця перевірка не відтворюється локально через `--dry-run`.

## Considered Options
* Перейменувати під власний скоуп: `@vitaliytv/n7`
* Використати unscoped ім'я: `n7-cli`
* Лишити скоуп `@7n`, змінити ім'я пакета: `@7n/cli`

## Decision Outcome
Chosen option: "`@7n/cli`", because орг `7n` вже належить користувачеві (підтверджено скріншотом з npmjs.com), `@7n/cli` вільне (перевірено через `npm view @7n/cli` → E404), і назва `cli` узгоджується з рішенням про перейменування bin-файлу.

### Consequences
* Good, because `npm publish --dry-run` після зміни показав `name: @7n/cli`, `filename: 7n-cli-0.1.0.tgz`, `public access` — без помилок.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Змінені поля в `npm/package.json`:
```json
"name": "@7n/cli",
"description": "CLI-утиліта @7n/cli"
```
Перевірка доступності: `npm view @7n/cli version` → `npm error 404 Not Found`.
Перевірка приналежності орг: `npm access list packages @7n` — порожньо (жодного раніше опублікованого пакету), але орг підтверджена через UI npmjs.com.
