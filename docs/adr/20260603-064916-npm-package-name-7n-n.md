# Назва npm-пакета: `@7n/n` замість `@7n/7`

**Status:** Accepted
**Date:** 2026-06-03

## Context and Problem Statement

Спроба `npm publish` пакета з `"name": "@7n/7"` завершилась помилкою `E400 Bad Request — That word is not allowed` — ім'я `7` знаходиться в серверному денлисті npmjs.com. Ця перевірка не відтворюється через `--dry-run`, тому локально помилка не виявляється заздалегідь.

## Considered Options

* `@7n/7` — оригінальна назва, відхилена денлистом npm (E400)
* `@7n/cli` — описова назва, вільна на реєстрі (`npm view` → E404)
* `@7n/n` — коротка однолітерна назва, вільна на реєстрі (`npm view` → E404)

## Decision Outcome

Chosen option: "`@7n/n`", because цифрова назва `7` відхиляється серверним денлистом npm (E400), а `@7n/n` вільне на реєстрі, скоуп `@7n` належить користувачу, і реальна публікація підтвердила прийнятність назви.

### Consequences

* Good, because реальний `npm publish` повернув `+ @7n/n@0.1.0` без помилок — денлист пропустив назву.
* Bad, because денлист npm перевіряється лише при реальному `PUT` — `--dry-run` не відтворює `E400`, тому остаточне підтвердження назви можливе лише через реальну публікацію.

## More Information

* Файл: `npm/package.json`, поле `name`: `"@7n/n"`
* Скоуп `@7n` — активна npm-організація, підтверджена через npmjs.com Account Settings; `npm whoami` → `vitaliytv`
* Перевірка доступності: `npm view @7n/n version` → `E404 Not Found` (вільне до публікації)
* Реальна публікація: `+ @7n/n@0.1.0`
* `publishConfig.access: "public"` потрібне окремо для уникнення `E402` (scoped-пакети за замовчуванням приватні)
