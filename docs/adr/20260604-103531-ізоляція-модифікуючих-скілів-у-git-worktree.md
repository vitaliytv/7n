## ADR Ізоляція модифікуючих скілів у git-worktree

## Context and Problem Statement
Скіл `n-fix` вносить зміни в репозиторій і при виконанні в основному дереві може конфліктувати з незакомітченими змінами або паралельною роботою. Виникає потреба забезпечити ізольоване середовище для таких скілів.

## Considered Options
* Виконання `n-fix` безпосередньо в основному дереві
* Виконання `n-fix` у відокремленому git-worktree через `npx @nitra/cursor worktree add`

## Decision Outcome
Chosen option: "Виконання `n-fix` у відокремленому git-worktree", because SKILL.md містить preflight-блок `n-cursor:worktree:start`, що забороняє запуск поза `.worktrees/`; при виявленні основного дерева агент автоматично створює worktree за конвенцією `<current-branch>-fix`.

### Consequences
* Good, because зміни ізольовані від основного дерева: `npx @nitra/cursor worktree add "main-fix" "n-fix: worktree-only skill"` створює `.worktrees/main-fix/` без ризику забруднити `main`.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
- Preflight: `git rev-parse --show-toplevel` + `git branch --show-current` → якщо не під `.worktrees/`, виконується `npx @nitra/cursor worktree add`.
