# getw: дельта-перенесення через `git apply` замість сліпого `git checkout -- .`

**Status:** Accepted
**Date:** 2026-06-03

## Context and Problem Statement

Команда `getw` (у `npm/getw.js`) переносила зміни з git-worktree у поточну гілку через `git checkout "$target_branch" -- .`, що замінювала весь вміст робочого дерева файлами з worktree-гілки. Файли, змінені лише у поточній гілці (але не в worktree-гілці), безмовно перезаписувалися старими версіями — без виявлення конфліктів і без можливості відновлення.

## Considered Options

- `git checkout "$target_branch" -- .` — сліпе перезаписування всього робочого дерева (попередня реалізація)
- Перенесення лише дельти worktree-гілки через `git diff --binary <merge-base> <target> | git apply --whitespace=nowarn`

## Decision Outcome

Chosen option: "Перенесення лише дельти worktree-гілки через `git diff | git apply`", because команда `git checkout <branch> -- .` затирала файли поточної гілки, не зачеплені worktree-гілкою; новий підхід обчислює спільного предка через `git merge-base`, генерує патч лише реально змінених файлів і безпечно зупиняється при конфлікті замість мовчазної втрати даних.

### Consequences

- Good, because файли, змінені лише у поточній гілці, більше не перезаписуються — `git apply` без `--index` чіпає виключно файли з патча.
- Good, because при конфлікті `git apply` повертає ненульовий код виходу, worktree зберігається для ручного злиття, а зміни поточної гілки лишаються недоторканими.
- Good, because зміни кладуться як unstaged (як і раніше очікує решта workflow).
- Bad, because якщо `git merge-base` не знаходить спільного предка (гілки не мають спільної історії), перенесення повністю скасовується.

## More Information

- Змінений файл: `npm/getw.js` (рядки 64–78)
- Змінений файл: `npm/README.md` (опис поведінки `getw` уточнено: «переносить лише дельту»)
- Change-файл: `npm/.changes/1780465944005-c49062.md` (bump `patch`, секція `Fixed`)
- Ключові команди у новій реалізації:
  ```sh
  local merge_base=$(git merge-base "$current_branch" "$target_branch")
  git diff --binary "$merge_base" "$target_branch" | git apply --whitespace=nowarn
  ```
- Прапор `--binary` у `git diff` — для коректної обробки бінарних файлів.
- Прапор `--whitespace=nowarn` у `git apply` — щоб відмінності у пробілах не спричиняли аварійне завершення.
- `git apply` викликається без `--index` — зміни лишаються unstaged, індекс не зачіпається.
- Тести (`npm/tests/index.test.mjs`) пройшли після змін; лінтер чистий.
