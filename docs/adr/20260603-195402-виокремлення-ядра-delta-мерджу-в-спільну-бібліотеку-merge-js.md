## ADR Виокремлення ядра delta-мерджу в спільну бібліотеку `merge.js`

## Context and Problem Statement
Механізм «інтелектуального» багатокрокового мерджу (Tier 0–3: `git apply` → `git merge-file --diff3` → `mergiraf` → LLM-агент) реалізований безпосередньо в `getw.js`. Виникла необхідність повторно використати цей самий механізм у новій команді `getpull`, щоб уникнути дублювання логіки.

## Considered Options
* Скопіювати zsh-логіку безпосередньо в `getpull.js`
* Винести спільний код у окремий модуль `merge.js`

## Decision Outcome
Chosen option: "Винести спільний код у окремий модуль `merge.js`", because користувач ініціював завдання саме як «винесемо в окремий lib», а повторна реалізація в двох файлах ускладнила б підтримку Tier-логіки.

### Consequences
* Good, because `npm/merge.js` експортує `MERGE_ZSH_LIB` (zsh-фрагмент із `_n7merge_delta` і tier-хелперами) та `runZsh` — єдина точка змін для всіх команд, що роблять delta-перенесення.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Файли: `npm/merge.js`, `npm/getw.js`, `npm/getpull.js`. Публічні exports: `MERGE_ZSH_LIB` (string), `runZsh(script, spawnFn, argv)`. Тести: `npm/tests/merge.test.mjs`. Валідація: `zsh -n` обох генерованих скриптів — OK, `vitest` — 29/29.

## ADR Delta-перенесення (merge-base..src) у `getpull` замість повного checkout

## Context and Problem Statement
Перша реалізація `getpull` використовувала `git checkout origin/$branch -- .`, що деструктивно перезаписує **всі** tracked-файли версією з origin — незалежно від наявності незакомічених правок. Під час тестування (реальний виклик `bun ./npm/bin/n.js getpull`) виявлено, що це безпечно лише коли origin вже попереду локального HEAD, але затирає незакомічені зміни в загальному випадку.

## Considered Options
* Повний `git checkout origin/$branch -- .` (перша реалізація)
* Delta-перенесення через `_n7merge_delta "HEAD" "origin/<branch>"` (ті самі Tier 0–3, що й `getw`)

## Decision Outcome
Chosen option: "Delta-перенесення через `_n7merge_delta`", because той самий недолік (деструктивний повний checkout) змусив переписати `getw` на delta-логіку раніше — користувач прямо сказав «хочу `getpull` до тієї ж delta-логіки».

### Consequences
* Good, because transcript фіксує очікувану користь: staged зміни (index) та локальні коміти не чіпаються взагалі; unstaged конфлікт виводить diff3-маркери і exit code 1 (fail-safe), вміст не втрачається; перевірено емпірично на тимчасовому репозиторії в `/tmp/gptest` і `/tmp/gpconf`.
* Bad, because при суто-unstaged конфліктному вмісті оригінал файлу зберігається **лише в маркерах** — окремого rollback-механізму немає (зафіксовано в transcript як залишковий ризик).

## More Information
Файли: `npm/getpull.js`, `npm/merge.js`. Команда: `bun ./npm/bin/n.js getpull [гілка]`. Тестовий сценарій: локальний коміт не з origin + `d.txt`/`f.txt` у stage + `a.txt`/`e.txt` unstaged + конфліктний `g.txt` → усі перевірки пройшли. Env `N7MERGE_NO_MERGIRAF=1` вимикає Tier 2 (mergiraf), але не LLM-агент.

## ADR Нейтральні env-змінні `N7MERGE_*` із backward-compat фолбеком на `GETW_*`

## Context and Problem Statement
Спільне ядро `merge.js` використовується і `getw`, і `getpull`, але env-змінні для налаштування моделі та mergiraf мали префікс `GETW_` (`GETW_MERGE_MODEL`, `GETW_MERGE_CURSOR_MODEL`, `GETW_NO_MERGIRAF`), що є командо-специфічним і неприйнятним для спільного lib.

## Considered Options
* Залишити `GETW_*`-префікс (первинна реалізація)
* Ввести нейтральні `N7MERGE_*` із backward-compat фолбеком на `GETW_*`

## Decision Outcome
Chosen option: "Нейтральні `N7MERGE_*` із backward-compat фолбеком", because користувач прямо запросив «хочу нейтральні».

### Consequences
* Good, because transcript фіксує очікувану користь: вкладений zsh-фолбек `${N7MERGE_MODEL:-${GETW_MERGE_MODEL:-sonnet}}` перевірено у zsh (пріоритет N7MERGE_ → GETW_ → дефолт); існуючі скрипти з `GETW_*` продовжують працювати без змін.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Файли: `npm/merge.js`. Конкретні змінні: `N7MERGE_MODEL` (фолбек `GETW_MERGE_MODEL`, дефолт `sonnet`), `N7MERGE_CURSOR_MODEL` (фолбек `GETW_MERGE_CURSOR_MODEL`), `N7MERGE_NO_MERGIRAF` (фолбек `GETW_NO_MERGIRAF`, дефолт `0`). Тест у transcript: `zsh -c` з трьома варіантами env → очікувана пріоритизація підтверджена.
