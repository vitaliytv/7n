# Нейтральні env-змінні `N7MERGE_*` із backward-фолбеком на `GETW_*`

**Status:** Accepted
**Date:** 2026-06-03

## Context and Problem Statement

Після виносу ядра мерджу в `merge.js` env-змінні для налаштування моделі та mergiraf мали префікс `GETW_` (`GETW_MERGE_MODEL`, `GETW_MERGE_CURSOR_MODEL`, `GETW_NO_MERGIRAF`). Оскільки ядро тепер спільне для `getw` і `getpull`, командо-специфічний префікс є семантично неточним і неприйнятним для спільної бібліотеки.

## Considered Options

- Залишити `GETW_*` (без змін)
- Ввести `N7MERGE_*` і видалити `GETW_*` (breaking change)
- Ввести `N7MERGE_*` із вкладеним фолбеком `${N7MERGE_X:-${GETW_X:-default}}` (backward-сумісно)

## Decision Outcome

Chosen option: "`N7MERGE_*` із вкладеним фолбеком на `GETW_*`", because користувач явно попросив нейтральні імена; вкладений фолбек зберігає backward-сумісність для наявних конфігурацій.

### Consequences

- Good, because `N7MERGE_MODEL` має пріоритет над `GETW_MERGE_MODEL`; якщо виставлено лише `GETW_MERGE_MODEL` — продовжує працювати; без жодного — дефолт `sonnet`. Підтверджено в zsh.
- Good, because `npm/tests/merge.test.mjs` містить спеціальний тест `'env-кнопки нейтральні (N7MERGE_*) із backward-фолбеком на GETW_*'`.
- Bad, because transcript не містить підтверджених негативних наслідків.

## More Information

- Env-змінні у `MERGE_ZSH_LIB` (`npm/merge.js`): `N7MERGE_MODEL` / `GETW_MERGE_MODEL` (дефолт `sonnet`), `N7MERGE_CURSOR_MODEL` / `GETW_MERGE_CURSOR_MODEL`, `N7MERGE_NO_MERGIRAF` / `GETW_NO_MERGIRAF` (дефолт `0`).
- Перевірка: `zsh -c 'echo "${N7MERGE_MODEL:-${GETW_MERGE_MODEL:-sonnet}}"'` з трьома комбінаціями env.
- Оновлено: `npm/README.md`, JSDoc у `npm/getw.js` і `npm/getpull.js`.

## Update 2026-06-03

Додаткова верифікація пріоритизації: `N7MERGE_MODEL=haiku` перемагає `GETW_MERGE_MODEL=opus`; без `N7MERGE_MODEL` — `GETW_MERGE_MODEL=opus` застосовується; без обох — `sonnet`. Документацію і JSDoc оновлено в `npm/getw.js`, `npm/getpull.js`, `npm/README.md`.
