## ADR Сквошинг усіх локальних комітів і змін робочого дерева в один коміт під час `push`

## Context and Problem Statement
У проєкті існує команда `push` (`npm/push.js`), яка об'єднує локальний стан репозиторію перед надсиланням до remote. Питання полягало в тому, чи враховує `push` лише staged-зміни чи й ширший контекст (незакомічені зміни, untracked-файли, наявні локальні коміти).

## Considered Options
* Сквошити всі локальні коміти (`origin/<branch>..HEAD`) плюс усі зміни робочого дерева (`git add -A`) в один коміт
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Сквошити всі локальні коміти (`origin/<branch>..HEAD`) плюс усі зміни робочого дерева (`git add -A`) в один коміт", because коментар у `npm/push.js` прямо фіксує цю поведінку: zsh-функція `push` бере ВСІ локальні коміти (`origin/<branch>..HEAD`) і ВСІ зміни робочого дерева (staged/unstaged/untracked через `git add -A`) та сквошить їх в ОДИН коміт на вершині.

### Consequences
* Good, because transcript фіксує очікувану користь: remote завжди отримує єдиний, чистий коміт без проміжної «брудної» історії.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
- Реалізація: `npm/push.js` (імпортує `MERGE_ZSH_LIB`, `runZsh` з `npm/merge.js`)
- Тести: `npm/tests/push.test.mjs`
- Зміни робочого дерева додаються через `git add -A` перед сквошингом
