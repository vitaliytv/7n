## ADR Виправлення getw: безпечне перенесення дельти worktree-гілки без затирання локальних змін

## Context and Problem Statement
Команда `getw` використовувала `git checkout "$target_branch" -- .`, що безумовно затирала весь вміст робочого дерева поточної гілки файлами з worktree-гілки, включно з файлами, які worktree-гілка не чіпала. Це спричиняло мовчазну втрату незакомічених або локальних змін у поточній гілці.

## Considered Options
* `git checkout <branch> -- .` (попередня реалізація — сліпе перезаписування всього робочого дерева)
* `git diff --binary <merge_base> <target> | git apply` (перенесення лише дельти від спільного merge-base)

## Decision Outcome
Chosen option: "перенесення дельти через `git diff | git apply`", because такий підхід не чіпає файли, які worktree-гілка не змінювала, і безпечно падає при конфлікті (замість мовчазного затирання).

### Consequences
* Good, because файли, змінені лише в поточній гілці, більше не перезаписуються.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Файл: `npm/getw.js`. Ключова заміна: рядок 63 `git checkout "$target_branch" -- .` → `git diff --binary "$merge_base" "$target_branch" | git apply --whitespace=nowarn`.

## ADR Обробка конфліктів у getw: git apply --3way → пофайловий git merge-file

## Context and Problem Statement
Після першого виправлення (`git apply`) команда `getw` падала при конфлікті. Спроба `git apply --3way` завершувалась помилкою `does not match index` на dirty робочому дереві (обидві гілки незалежно правили ті самі tooling-файли), а атомарність `git apply` не дозволяла частково застосувати патч.

## Considered Options
* `git apply --3way` (завершується помилкою `does not match index` при dirty worktree)
* `git apply --reject` (неатомарний, кладе `.rej`-файли, делегує їх видалення агенту)
* Пофайловий `git merge-file --diff3` (без індексу, неатомарний, сам генерує diff3-маркери)

## Decision Outcome
Chosen option: "пофайловий `git merge-file --diff3`", because працює лише по файлах без звернення до git-індексу, що повністю усуває причину помилки `does not match index`; детерміністичний вердикт скрипта (grep маркерів) не залежить від поведінки агента.

### Consequences
* Good, because скрипт сам виносить вердикт успіху/невдачі через grep конфліктних маркерів, агент лише прибирає маркери.
* Good, because `--diff3` зберігає base-секцію (`|||||||`), яку використовує mergiraf на наступному рівні.
* Bad, because `git merge-file` може позначати конфліктними суміжні (не лише перетинні) зміни — стандартна поведінка diff3.

## More Information
Файл: `npm/getw.js`. Функції `_getw_files_with_markers`, `_getw_resolve_with_agent`. Функціональний тест виконано у тимчасовому git-репозиторії (`/tmp/getw_func/run.sh`). Раніше обговорена альтернатива `--reject` відкинута, бо вимагала делегування `rm` агенту, що є ненадійним сигналом завершення.
