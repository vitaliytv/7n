## ADR Видалення префікса `./` у значенні `bin` у `package.json`

## Context and Problem Statement

При виконанні `npm publish` реєстр npm автоматично прибирав запис `bin` з `package.json` і виводив попередження `"bin[...] script name ... was invalid and removed"`. Початкова гіпотеза — цифра в імені ключа або назві файлу — не підтвердилась.

## Considered Options

* Значення `bin` із префіксом `"./bin/cli.js"`
* Значення `bin` без префікса `"bin/cli.js"`

## Decision Outcome

Chosen option: "значення `bin` без префікса `\"bin/cli.js\"`", because емпіричний dry-run показав: `"cli": "./bin/cli.js"` (з `./`) викидається, а `"cli": "bin/cli.js"` (без `./`) проходить без попереджень. `npm pkg fix` цю проблему не ловить і в попередженні вводить в оману.

### Consequences

* Good, because `npm publish --dry-run` виходить чистим без жодного `auto-corrected` попередження, що підтверджено transcript.
* Bad, because `npm pkg fix` лишає `./` як є, тому майбутні редагування `package.json` можуть ненавмисно повернути префікс.

## More Information

Зміна в `npm/package.json`: `"7n": "./bin/cli.js"` → `"7n": "bin/cli.js"`.
Емпіричне підтвердження через temp-директорії з варіантами ключів і значень у dry-run.

## ADR Перейменування bin-entrypoint та команди CLI

## Context and Problem Statement

Файл `npm/bin/7.js` і команда `"7"` в `bin` потребували перейменування — частково через підозру щодо валідації npm, частково за явним запитом користувача змінити ім'я файлу на `cli.js`.

## Considered Options

* Залишити `bin/7.js`, команда `"7"` — початковий стан
* Команда `"7n"`, файл `bin/7.js` — проміжний варіант, обраний «варіантом 2» (ім'я файлу не важливе, важливий ключ)
* Перейменувати файл на `bin/cli.js`, команда `"cli"` — остаточне рішення за запитом користувача

## Decision Outcome

Chosen option: "перейменувати файл на `bin/cli.js`, команда `\"cli\"`", because користувач явно запросив: "давай 7.js перейменуємо в cli.js" і потім "перейменуй в cli також" (команду). Файл містить лише 4 рядки-проксі до `index.js`, тому перейменування безпечне.

### Consequences

* Good, because назва файлу та назва команди збігаються — простіше орієнтуватися.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information

Файли: `npm/bin/7.js` → `npm/bin/cli.js`.
Оновлено у `npm/package.json`: ключ `"bin"` і скрипт `"start": "bun ./bin/cli.js"`.

## ADR Додавання `publishConfig.access: "public"` для скоупованого пакета

## Context and Problem Statement

`npm publish @7n/n` завершився помилкою `E402 Payment Required` — scoped-пакети (`@scope/...`) npm за замовчуванням трактує як приватні, а приватна публікація потребує платного плану.

## Considered Options

* Разовий флаг `npm publish --access public`
* Поле `"publishConfig": { "access": "public" }` у `package.json`

## Decision Outcome

Chosen option: "поле `\"publishConfig\": { \"access\": \"public\" }` у `package.json`", because це закріплює налаштування в репо і не вимагає пам'ятати про флаг при кожній публікації, у т.ч. в CI.

### Consequences

* Good, because dry-run після зміни: `Publishing to https://registry.npmjs.org/ with tag latest and public access` — помилка E402 більше не виникає.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information

Зміна в `npm/package.json`:

```json
"publishConfig": { "access": "public" }
```

## ADR Перейменування пакета `@7n/7` → `@7n/n`

## Context and Problem Statement

`npm publish` повернув `E400 Bad Request – That word is not allowed` для назви `@7n/7` — серверний денлист npm відхиляє ім'я пакета. Скоуп `@7n` валідний (організація `7n` належить користувачу).

## Considered Options

* `@7n/7` — початкова назва, відхилена денлистом
* `@7n/cli` — проміжний варіант, вільне ім'я, dry-run чистий; відхилений користувачем на користь коротшого
* `@7n/n` — літерне односимвольне ім'я; перевірено через `npm view @7n/n` (E404 — вільне); запит пройшов реєстрацію після публікації `@7n/n@0.1.0`
* Інші варіанти: `@vitaliytv/n7`, `n7-cli`, `@7n/7` із символом (наприклад `_7`) — запропоновані, але не обрані

## Decision Outcome

Chosen option: "`@7n/n`", because цифрова назва `7` відхиляється денлистом npm (E400), а односимвольна літера `n` проходить; скоуп `@7n` зберігається, що відповідає бажанню залишити бренд `7n`.

### Consequences

* Good, because реальний `npm publish` повернув `+ @7n/n@0.1.0` без помилок — денлист пропустив назву.
* Bad, because пакет установлює команду `cli`, а не `n` або `7` — назва пакета і назва CLI-команди не збігаються (обговорювалось, але не виправлялось у цій сесії).

## More Information

Зміна в `npm/package.json`: `"name": "@7n/7"` → `"name": "@7n/n"`.
Підтверджено: `npm view @7n/n` → E404 (вільне) до публікації; `npm publish` → `+ @7n/n@0.1.0` після.
Коміт: `c729d26 npm: публікація @7n/n — фікс bin, scope-name і public access`.
