# Лаконічний тірований вивід delta-мерджу у `getw`/`pull`

**Status:** Accepted
**Date:** 2026-06-04

## Context and Problem Statement

`getw` і `pull` використовують спільне ядро `_n7merge_delta` (`npm/merge.js`). При Tier 1 (clean git apply) та Tier 2 (mergiraf) вивід містив побічні повідомлення: `error: patch failed`, `error: … patch does not apply`, а також пофайлові `mergiraf розв'язав …`. Ці повідомлення лякали користувача навіть при успішному завершенні і не несли actionable інформації.

## Considered Options

* Показувати лаконічний тірований підсумок замість пофайлового шуму; деталі лише для Tier 3 LLM
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome

Chosen option: "Показувати лаконічний тірований підсумок замість пофайлового шуму", because користувач явно вказав бажаний формат: рядки `error:` мають бути приховані, деталі mergiraf — заховані, а резюме складається з трьох рядків (Tier 1 / Tier 2 / Tier 3) з per-file блоками лише для Tier 3.

### Consequences

* Good, because transcript фіксує очікувану користь: false-alarm `error:` повідомлення зникають при успішному мерджі.
* Good, because Tier 3 показує структуровані блоки конфлікту (ours / src / result) разом із LLM thinking, що дає контекст без шуму проміжних кроків.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information

- Реалізація: `npm/merge.js` — функції `_n7merge_delta` і `_n7merge_resolve_with_agent` (4-й параметр `summary_out`).
- Tier 1 шум приховується через `2>/dev/null` при `git apply`; Tier 2 — через лічильник замість `echo "mergiraf розв'язав …"`.
- Тести: `npm/tests/merge.test.mjs` (55 тестів, усі зелені після змін).
- Change-файл: `.changes/1780559809432-7d3153.md` (bump `minor`, секція `Changed`).
- Формат підсумку, заданий користувачем:

```
Tier 1 (git):      X файл(ів)
Tier 2 (mergiraf): Y файл(ів)
Tier 3 (LLM):      Z файл(ів)

📄 <filename>
── Приймач (поточна …): …
── Джерело (src): …
── Результат: …
🤖 <LLM thinking>
```
